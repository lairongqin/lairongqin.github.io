ATOM

最近ATOM很火，就尝试着想用一用看一看，具体各种基本操作谷歌或者百度一大把，说一下我遇到的坑吧。

package

系统环境是win7。

于是欢快的从atom.io上下载安装包安装好后，发现最重要的功能package无法内置安装！！

那还有什么用！说好的sublime呢。

于是乎从ERRO问题入手，一连串英语，”….c++…vs…”大概意思就是要安装个VS来使C++可以编译吧。

看来要安装VS环境来编译C++，好大，不想安！于是发现解决办法

解决办法一

跑到.atom/packages/的文件夹，用git 把包的仓库地址clone下来，然后用npm install，安装，安装结束重启ATOM就能用了！

缺点：麻烦！跟atom内置安装的相比，不顺畅。反正各种差，还是放弃这种方法吧

解决办法二

老老实实安包吧，亲 地址在这 很小的一个包。原来根本不用安装VS那么大的东西！！！

eslint

好哒，现在可以安装eslint了。

先在setting的package里面搜到linter，下载量最多的那个，接着安装linter-eslint

这两个package用来支持eslinter，只是一个工具包。

最后在 npm install eslint --save-dev 来下载eslint。

在安装完毕后在moudlue的bin目录下有一个可执行文件eslint，使用命令 eslint –init
就可以初始化了。

eslint安装完毕后，可以在ATOM的linter-eslint的配置选项中，选择目录位置。
但是推荐大家把init后生成的eslintrc文件直接拷贝到工程文件根目录下，因为eslint的规则是匹配检验包含eslint的文件目录和子目录中的JS文件。
