    • 基于当前数组中的所有项创建一个新数组，拼接参数返回构建的数组。

  • splice()

迭代方法

  • every()：对数组每一项运行给定的函数，如果对每一项都返回true，则返回true。
  • filter()：对数组的每一项运行给定的函数，返回该函数会返回true的项组成的数组。
  • forEach()：对数组中的每一项运行给定的函数，该方法没有返回值。
  • map()：对数组中的每一项运行给定函数，返回每次函数调用结果组成的数组。
  • some()：对数组中的每一项运行给定函数，如果该函数对任一项返回true，则返回true。

Date类型

要创建一个Date类型，使用new操作符即可。

var now = Date();

在调用Date构造函数而不传参数的情况下，新创建的对象自动获得当前的事件和日期。

  • Date.parse()
    传日期字符串，尝试解析，返回毫秒数
    如果直接将表示日期的字符串传递个Date构造函数，也会在后台调用Date.parse()
  • Date.UTC()
    • 该方法返回日期的毫秒数。
    • Date.UTC(年份，基于0的月份，月份中哪一天，小时数，分钟，秒，毫秒数)，前两个参数必须，第三个参数默认缺省为1，剩下的默认缺省为0
  • Date.now()
    返回调用这个方法时的日期和时间的的毫秒数。

继承的方法

  • toLocaleString()
  • toString()
  • valueOf()

因为valueOf返回毫秒数，所以我们可以直接比较两个日期的大小。

日期格式化方法

  • toDateString() 以特定于实现的格式显示星期几、月、日和年。
  • toTimeString() 以特定于实现的格式显示时、分、秒和时区
  • toLocaleDateString() 以特定于地区的格式显示星期几、月、日和年
  • toLocaleTimeString() 以特定于实现的格式显示时、分、秒
  • toUTCString() 以特定于现实的格式完整的UTC日期

RegExp

正则表达式创建
: var expression = / pattern /flags ;

  • flag
    • g 全局模式
    • i 不区分大小写
    • m 多行模式

正则表达式创建
: var pattern = new RegExp(“匹配字符串模式”,”标志字符串”);

实例属性

  • global:布尔值，表示是否设置了g标志
  • ignoreCase：布尔值，表示是否设置了i标志
  • lastIndex：整数，表示开始搜索下一个匹配项的字符位置，从0算起。
  • multiline：布尔值，表示是否设置了m标志。
  • source:正则表达式的字符串表示

RegExp实例方法

正则表达式30分钟入门

  • exec()；
    • 接收一个参数，即要匹配的字符串，返回包含第一个匹配项信息的数组；或者在没有匹配的情况下返回null。
    • 返回的数组虽然是Array的实例，但包含额外两个属性：index和input。其中，index表示匹配项在字符串中的位置，input表示应用正则表达式的字符串。
    • 对于exec()方法而言，即使在模式中设置了全局标志（g），它每次也只会返回一个匹配项。在不设置全局标志的情况下，在同一个字符串上多次调用exec()将始终返回第一个匹配的信息。而在设置全局标志的情况下，每次调用exec()则都会在字符串中继续查找匹配项。

  • test()

    • 接受一个字符串参数。
    • 返回true或false

  • toLocaleString()和toString()方法都会返回一个字符串，返回正则表达式的字面量形式。

RegExp构造函数属性

RegExp 构造函数包含一些属性。这些属性作用于域中的所有正则表达式，并且基于所执行的最后一次正则表达式操作而变化。可以有两种方式访问它们。

长属性名 短属性名 说明
input \$_ 最近一次要匹配的字符串
lastMatch \$& 最近一次的匹配项
lastParen \$+ 最近一次匹配的捕获组
leftContext input字符串中lastMatch之前的文本
multiline \$* 布尔值，表示是否用多行模式
rightContext \$` input字符串中lastMatch之后的部分
\$1……\$9 储存捕获组

基本包装类型

  • 引用类型与基本包装类型的主要区别就是对象的生存期。
    • 使用new操作符创建的引用类型的实例，在执行流离开当前作用域之前都一直保存在内存中。
    • 自动创建的基本包装类型的对象，则只存在于一行代码的执行瞬间，然后立即被销毁。
