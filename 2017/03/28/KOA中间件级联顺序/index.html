Koa中间件级联顺序

Koad的中间件使用一种更加传统的方式进行级联，通过async函数来实现（Promise跟yield的语法糖），放弃了曾经不友好的callback回调方式。无论是语法糖还是如何，我们通过async函数可以实现“真实”的中间件。相较于Connect的直接把控制权通过一系列函数来处理直到其中一个返回结果的实现方法来说，Koa通过async函数把控制权交给下游中间件（“downstream”）后还可以交还给上游的中间件（“upstream”）。

上面的例子返回了一个单纯的“hello word网页”，首先request请求通过A中间件，控制台输出“A 1”当遇到next()语句时，便把控制权交给了接下来的B中间件，以此类推，一直到最后的中间件完成后，栈就会释放，每个中间件依次重新得到控制权，进行“upstream”的行为，即又返回C -> B -> A 的顺序。

这就是Koa中间件的级联顺序。
