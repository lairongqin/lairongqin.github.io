检查浏览器支持性

我们通过检测 Navigator上的 serviceWorker 的属性和 window 上的 PushManager属性来检查浏览器的支持性,通过navigatoer.serviceWorker.register() 方法来注册service Worker 。

订阅服务 subscribe

在订阅推送服务时，我们需要向后端发送两个东西，一个是 push message 的 endpoint ,一个是 key 。endpoint是辨别订阅会话所用。key是公钥用来进行数据加密。配套网站 ， 或者使用这其中的函数库。

订阅服务，我们需要的到用户允许 push message的权限。通过 Notification.permission() 方法。

订阅推送服务主要使用了 ServiceWorkerRegistration.pushManager.subscribe() 方法，该方法接受一个参数对象，userVisibleOnly跟applicationServerKey。
userVisibleOnly: 一个布尔值，表示返回的push subscription只能用来处理消息，产生的效果让用户可见。
applicationServerKey: 一个公钥你的推送服务器将会用这个私钥来发送消息给客户端。

subscribe() 的成功返回需要两步：

  1. 用户授予展示通知的权限
  2. 浏览器已经发送了网络请求到push service去得到生成PushSubscription的细节。

检查是否订阅成功

检查订阅是否成功，我们主要使用了一个 ServiceWorkerRegistration.pushManager.getSubscription()的方法。
如果没有订阅任何 subscription 则返回null

监听push事件

push事件就是当用户接收到了一个push message所产生的事件。

当我们触发了一个push事件，浏览器接收到了push消息，唤醒这个push消息所对应的service worker并派发push事件。

self这个对象指代的是service worker本身，当一个push message被接收到时，事件监听被触发，我们通过showNotification 方法创建一个通知。
showNotification 方法接收两个参数，title和options

event.waitUntil() 接受一个Promise对象，并且保证service worker运行到接受的Promise对象被解决或者拒绝。

点击通知

当我们点击浏览器推送的通知的时候，你会发现没有反映。我们可以通过监听 notificationclick事件来解决这个问题。
